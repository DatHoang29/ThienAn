## Brief overview
- Project-specific guideline for moving and refactoring code: when a variable/config/source block is moved, move the related functions that operate on it as well.
- Prefer cohesive encapsulation over leaving tightly coupled variables and functions split across distant parts of the file.

## Refactoring workflow
- When introducing a constant, generated source string, config object, or shared helper, place it in the smallest scope that uses it.
- If a variable is used by exactly one function, move that variable inside the function instead of keeping it at module scope.
- If a function exists only to consume a specific variable, move both together as one logical block.
- Avoid refactors where a variable is moved to the top of the file while its only consumer remains far below without a clear reason.
- For Expo config plugins, keep generated native source strings inside the plugin function that writes them unless reused by multiple plugin functions.

## Encapsulation rules
- Treat related variables and functions as a small unit: source data, path builders, write/copy logic, and registration code should be easy to read as one flow.
- Prefer function-local constants for single-consumer generated content, such as Swift/Kotlin source strings generated by Expo plugins.
- Prefer small named helper functions when repeated path construction or file-generation logic appears in multiple places.
- Do not over-split single-purpose plugin logic into extra files unless the code becomes large or reused by multiple plugins.
- Keep comments short and technical, explaining why the block is grouped when the relationship is not obvious.

## Example trigger cases
- If `audioSessionConfigSwift` is only used by `withAudioSessionConfig`, declare it inside `withAudioSessionConfig`.
- If a plugin generates a native file, keep the generated source, destination path logic, and `fs.writeFileSync` flow together.
- If a constant is only used by one function, do not leave it separated from that function unless required by module initialization or reused elsewhere.
